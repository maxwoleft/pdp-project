# Потік скасування та перенесення записів через AI чат

## Огляд

AI чат автоматично розпізнає наміри клієнта скасувати або перенести запис та допоможе з цим процесом.

## Ключові слова для розпізнавання

### Скасування
- Українська: "скасувати", "скасувати запис", "відмінити", "відмінити запис"
- Російська: "отменить", "отменить запись"
- Англійська: "cancel", "cancel appointment"

### Перенесення
- Українська: "перенести", "перенести запис", "змінити час", "змінити дату"
- Російська: "перенести запись", "изменить время"
- Англійська: "reschedule", "change time", "change date"

## Потік обробки

### 1. Скасування запису

#### Крок 1: Клієнт просить скасувати
```
Клієнт: Хочу скасувати запис
```

#### Крок 2: AI пропонує перенести замість скасування
```
AI: Розумію, що у вас змінилися плани. Можливо, краще перенести запис на інший зручний час? 
    У нас є вільні слоти завтра та післязавтра.
```

#### Крок 3a: Якщо клієнт погоджується перенести
```
Клієнт: Так, давайте перенесемо
AI: [Переходить до потоку перенесення]
```

#### Крок 3b: Якщо клієнт наполягає на скасуванні
```
Клієнт: Ні, все одно треба скасувати
AI: Добре, допоможу скасувати. Для цього мені потрібен ваш номер телефону або email.
```

#### Крок 4: Збір контактних даних (якщо немає в профілі)
```
Клієнт: +380971234567
AI: Дякую! Шукаю ваші записи...
```

#### Крок 5: Показ записів та вибір
```
AI: Знайшла такі ваші записи:
    1. 25 січня 18:00 - Манікюр з Маріанною
    2. 28 січня 15:00 - Стрижка з Оленою
    
    Який запис скасувати?
```

#### Крок 6: Підтвердження
```
Клієнт: Перший
AI: Підтверджуєте скасування запису на 25 січня о 18:00 (Манікюр з Маріанною)?
```

#### Крок 7: Виконання скасування
```
Клієнт: Так
AI: ✅ Запис скасовано. Сподіваємось побачити вас пізніше!
```

### 2. Перенесення запису

#### Крок 1: Клієнт просить перенести
```
Клієнт: Можна перенести мій запис?
```

#### Крок 2: Збір контактних даних (якщо немає в профілі)
```
AI: Звичайно! Для цього мені потрібен ваш номер телефону або email.
Клієнт: +380971234567
```

#### Крок 3: Показ записів
```
AI: Знайшла такі ваші записи:
    1. 25 січня 18:00 - Манікюр з Маріанною
    2. 28 січня 15:00 - Стрижка з Оленою
    
    Який запис перенести?
```

#### Крок 4: Вибір нової дати та часу
```
Клієнт: Перший
AI: На яку дату та час вам зручно перенести?
```

#### Крок 5: Перевірка доступності
```
Клієнт: На завтра о 15:00
AI: Перевіряю доступність Маріанни на завтра о 15:00... 
    ✅ Маріанна вільна! Перенести запис?
```

#### Крок 6: Підтвердження та виконання
```
Клієнт: Так
AI: ✅ Запис перенесено на 26 січня о 15:00. Чекаємо на вас!
```

## Розумний контекст

AI розуміє контекст і може фільтрувати записи:

### Приклад 1: Клієнт вже уточнив послугу
```
Клієнт: Хочу скасувати запис на манікюр
AI: [Шукає контакти] → [Показує ТІЛЬКИ записи на манікюр]
```

### Приклад 2: Клієнт вже уточнив майстра
```
Клієнт: Перенести запис до Маріанни
AI: [Шукає контакти] → [Показує ТІЛЬКИ записи до Маріанни]
```

### Приклад 3: Клієнт вже уточнив дату
```
Клієнт: Скасувати запис на завтра
AI: [Шукає контакти] → [Показує ТІЛЬКИ завтрашні записи]
```

## Пошук без контактів

Якщо клієнта не знайдено по телефону/email, AI запитає додаткові дані:

```
AI: На жаль, не знайшла записів на цей номер. 
    Підкажіть, будь ласка:
    - Дату запису
    - Час запису
    - Послугу
    - Майстра
    
    Це допоможе знайти ваш запис.
```

## Технічна реалізація

### Функції в `appointment_helper.py`

```python
# Пошук клієнта
find_client_by_contact(database_code, phone=None, email=None)

# Отримання записів клієнта
get_client_appointments(database_code, client_id, from_date=None)

# Скасування
cancel_client_appointment(database_code, appointment_id, reason)

# Перенесення
reschedule_client_appointment(database_code, service_id, new_date, new_time, new_master_id=None)
```

### Збереження контексту в `context.user_data`

```python
# Для відстеження процесу
context.user_data["pending_cancel_or_reschedule"] = "cancel" | "reschedule"

# Знайдені записи
context.user_data["client_appointments"] = [...]

# Обраний запис для операції
context.user_data["selected_appointment"] = {...}
```

## Правила для AI

1. **Завжди пропонувати перенесення замість скасування**
2. **Не питати про інформацію, яку клієнт вже надав**
3. **Фільтрувати записи за контекстом (послуга/майстер/дата)**
4. **Підтверджувати перед виконанням операції**
5. **Давати чіткий фідбек про результат**

## Приклади діалогів

### Швидке скасування (з контекстом)
```
Клієнт: Скасувати запис на манікюр до Маріанни завтра
AI: [Має телефон з профілю] → [Знаходить запис] → 
    Підтверджуєте скасування запису на 26 січня о 18:00 (Манікюр з Маріанною)?
Клієнт: Так
AI: ✅ Запис скасовано
```

### Перенесення з пропозицією альтернатив
```
Клієнт: Перенести на завтра о 15:00
AI: На жаль, Маріанна зайнята завтра о 15:00. 
    Є вільні слоти:
    - Завтра о 14:00, 16:00, 17:00
    - Післязавтра о 15:00
    
    Який час вам підходить?
```

## Що далі

Можливі покращення:
1. Автоматичне визначення телефону з повідомлення
2. Розпізнавання дат у природній мові ("завтра", "наступного тижня")
3. Пропозиція альтернативного часу при конфлікті
4. Історія скасувань для аналітики
5. Штрафи за пізнє скасування (якщо потрібно)
