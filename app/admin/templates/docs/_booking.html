{# Онлайн запис (booking) #}

Що це

Pied de Poule Booking — окремий веб-застосунок самозапису для клієнтів салонів. Працює паралельно з AI чатом і використовує ту ж базу даних (per-country PostgreSQL). Інтеграція з AIHelps CRM для реальних слотів і створення записів.

Доступ для клієнтів

  • • Домен: https://booking.p-de-p.com
  • • Country picker на головній — клієнт обирає UK / UA / PL.
  • • Salon picker — багаторівневий dropdown по містах і конкретних салонах.
  • • Deep link до конкретного салону: https://booking.p-de-p.com/{salon-code}/{lang} (наприклад /l1/UA).
  • • AI чат банер у застосунку: натискання → https://ai-chat.p-de-p.com/chat/start?salon_id={uuid} — клієнт одразу потрапляє в AI чат свого салону.

Адмінка booking

  • • URL: https://booking.p-de-p.com/admin
  • • Логін: bookadm
  • • Пароль: зберігається у менеджера / у /opt/pdpbooking/app/.env (BOOKADM_PASSWORD)

Адмінка має дві ролі:

  • • admin — менеджер конкретного салону: redакція послуг, перегляд статистики, увімкнення/вимкнення AI чату.
  • • superadmin — повний доступ: створення салонів, локацій, користувачів, платіжних систем; bulk-toggle AI-чат банера на всіх салонах.

Технологічно

  • • Стек: Python 3.12 + Flask + gunicorn (3 workers) під systemd (pdpbooking.service).
  • • Хостинг: той же DigitalOcean droplet що й AI чат (167.172.100.158).
  • • Шлях: /opt/pdpbooking/app/; gunicorn слухає 127.0.0.1:5050.
  • • Nginx: reverse-proxy booking.p-de-p.com → 5050 + Let's Encrypt SSL.
  • • База даних: та сама PostgreSQL що й AI чат. Booking читає booking.salons, {country}.service, {country}.employee.
  • • CRM integration: AIHelps API через appointmentService.py — створення записів, freetime, employees.
  • • Frontend: Jinja2 + vanilla JS + інтeрактивний DOM (без React/Vue).
  • • Cache busting: mtime-based query params автоматично для CSS/JS.

Як AI чат і Booking працюють разом

  1. 1. Клієнт у Booking бачить банер "Зробіть замовлення через AI чат" вгорі сторінки салону.
  2. 2. Натискає → переходить на /chat/start?salon_id=<uuid> — AI чат автоматично починає розмову з контекстом цього салону.
  3. 3. AI чат підбирає послугу, майстра, час → створює запис через AIHelps API → той самий CRM що використовує Booking.
  4. 4. Запис зʼявляється у CRM як від Booking, так і від AI чату — менеджер бачить єдиний потік.

Toggle банера AI чату — у адмінці Booking → Салони → Edit → "AI чат". Bulk-увімкнення для всіх салонів — superadmin only.
