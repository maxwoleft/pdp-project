{# Booking: Керування #}

Дашборди

Дашборди — статистика записів за період + фільтри по салонах. Показує кількість записів, конверсію, топ-послуг.

Коли використовувати

  • Щоденний перегляд активності — скільки клієнтів записалося.
  • Аналіз популярних послуг — топ-5 за тиждень/місяць.
  • Виявлення проблем — раптовий спад записів = алярм.

Послуги

Послуги — каталог послуг конкретного салону + категорії. Послуги синхронізуються з AIHelps CRM автоматично. У адмінці можна:

  • Редагувати описи (показуються клієнту на сторінці запису)
  • Завантажувати фото послуги
  • Налаштовувати рекомендації (cross-sell після обраної послуги)
  • Архівувати непотрібні послуги (приховує з UI без видалення з CRM)

Коли втручатися вручну

  • CRM повернула криві назви — переписати у адмінці description
  • Хочеться рекламувати конкретні послуги — додати їх до рекомендацій
  • Сезонні offer-и — додати/прибрати тимчасові послуги

Салони (Superadmin only)

Салони — створення/редагування салонів + їх контактних даних. Кожен салон має:

  • code — короткий URL-код (наприклад l1, kyiv-obolon)
  • database_code — ID у AIHelps CRM (для API запитів)
  • country, city, address_line
  • telegram_token + telegram_chat_id — для нотифікацій
  • whatsapp_phone, instagram_url — соцмережі
  • smtp_from_email — від кого email-підтвердження
  • ai_chat_enabled — toggle банера "AI чат" на сторінці салону
  • payment_enabled — увімкнути EasyTip передоплату
  • status — active / planned (приховує "заплановані")

Bulk операції

  • AI чат bulk-toggle — увімкнути/вимкнути банер AI чату одночасно для всіх салонів.

Користувачі (Superadmin only)

Користувачі — облікові записи адмінів.

Ролі

  • superadmin — повний доступ до всіх салонів, налаштувань, користувачів. Bulk-операції.
  • admin — обмежений доступ: лише послуги + дашборди свого салону (передбачено через preset).

Як додати нового користувача

  1. Користувачі → "Додати"
  2. Логін + пароль + роль
  3. Зберегти. Користувач одразу може ввійти на /admin/login

Платіжні системи (Superadmin only)

Платіжні системи — налаштування EasyTip. Кожен салон може мати власні credentials платіжного провайдера.

  • Привязка локацій до payment system (1 локація = 1 system)
  • Конфіг credentials (api_key, location_id)
  • Toggle payment_enabled у налаштуваннях салону → активує передоплату

Як саме все працює разом

  1. Клієнт відкриває booking.p-de-p.com → обирає салон.
  2. Бачить категорії і послуги (із {country}.service у Postgres — синхронізованих з CRM щодня).
  3. Обирає послугу → майстра → слот (live з AIHelps API через get_freetime).
  4. Заповнює контакти → AI чат банер (опц., якщо клієнт хоче клопотатися через чат).
  5. Submit → POST /{salon}/create_appointment → AIHelps API booking → Telegram → email.
  6. Запис відображається у CRM салону одразу. Локальна копія у booking.appointmentsleads для аудиту.
