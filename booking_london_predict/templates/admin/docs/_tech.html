{# Booking: Опис технології #}

Що це таке

Pied de Poule Booking — веб-застосунок самозапису для клієнтів салонів краси. Клієнт обирає салон → послугу → майстра → час → залишає контакти → отримує запис у CRM. Працює паралельно з AI чатом і використовує ту ж базу даних.

Стек технологій

Backend

Python 3.12 + Flask + gunicorn (3 workers) під systemd. psycopg2-binary для PostgreSQL. MySQL-fallback для legacy.

Frontend

Jinja2 + vanilla JavaScript. Без React/Vue. Інтeрактивний DOM. Анімації CSS. intl-tel-input для phone input. Auto cache-busting через mtime.

База даних

PostgreSQL 16 — спільна з AI чатом. Таблиці: booking.salons, booking.appointmentsleads, {country}.service, {country}.employee.

Інтеграції

AIHelps CRM API — слоти, employees, створення записів. EasyTip — платежі. Telegram bot — нотифікації. SMTP — email.

Архітектура застосунку

Public-частина (клієнти)

  • GET / → перенаправлення на онбординг (вибір країни)
  • GET /{country} → список салонів країни
  • GET /{salon-code}/{lang} → сторінка салону (наприклад /l1/UA): категорії → послуги → майстри → слоти → форма даних
  • POST /{salon}/create_appointment → створення запису через AIHelps + Telegram + SMTP email
  • POST /reserve_slot / POST /release_slot → тимчасове блокування слоту перед оплатою

Admin-частина

  • GET /admin/login → форма входу
  • GET /admin/dashboard → статистика записів
  • GET /admin/services → каталог послуг + редагування описів/категорій
  • GET /admin/salons → керування салонами (superadmin only)
  • GET /admin/users → керування користувачами адмінки
  • GET /admin/payment_systems → платіжні системи

Інфраструктура

  • • Hosting: DigitalOcean droplet (167.172.100.158) — спільний з AI чатом
  • • Шлях: /opt/pdpbooking/app/
  • • Service: systemd pdpbooking.service
  • • Port: gunicorn слухає 127.0.0.1:5050
  • • Nginx: booking.p-de-p.com → 5050, з Let's Encrypt SSL
  • • Логи: /var/log/pdpbooking/access.log + error.log
  • • Cron: щоденний sync даних з AIHelps (через AI chat pipeline — спільна БД)

Flow запису клієнта

  1. Клієнт обирає країну → салон (multi-level dropdown по містах)
  2. Бачить категорії послуг у sidebar (Волосся / Манікюр / Брови / Подологія тощо)
  3. Обирає послугу; для деяких — додаткові параметри (довжина волосся, тип tier)
  4. Обирає майстра (або "будь-який доступний")
  5. Обирає день і час з вільних слотів (live з AIHelps)
  6. Заповнює контакти (ім'я + телефон з intl-tel-input)
  7. (Опц.) Здійснює передоплату через EasyTip
  8. POST create_appointment → AIHelps API → Telegram → email → запис у БД

Інтеграція з AI чатом

На сторінці кожного салону відображається банер "Зробіть замовлення через AI чат" (якщо salon.ai_chat_enabled = true). Натискання → ai-chat.p-de-p.com/chat/start?salon_id={uuid}. AI чат бронює слоти у тій же CRM — обидві системи бачать єдиний потік записів.
